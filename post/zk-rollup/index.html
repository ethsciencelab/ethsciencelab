Layer2 & zk rollup

2021-04-02 2 min read

ZK rollup 简介：

  • 以太坊网络拥堵，矿工费高，影响以太坊生态，尤其是 defi 的发展 -> layer 2
  • 有多种方法可以实现 Layer 2，zk rollup 是其中的一种
  • 可以把 zk rollup 想成一个链下的区块链（相对于主链），大家在 zk rollup 进行交易之后，会汇总成一份状态更新，然后用零知识证明的方式提交到主链
  • 对于主链的矿工来说，他只需要验证一份交易，就等于验证了很多笔交易，因此实现扩容
  • zksync 是 matter labs 推出的 zk rollup 产品。不同公司都可以推出自己的 zk rollup 方案，会产生好多个 zk rollup。

zk rollup signing key

  • 在以太坊主链会有一个智能合约，可以想成是放在 L1 的金库，在 L2 会有一个相对应的金库
    • 譬如我今天要使用 A 公司的 L2，我就要去 A 合约注册（L1-address, L2-public key)
    • L2-public key 只要符合 A 公司的格式要求就可以，不见得一定要按照他们提供的方法生成
  • zksync 提供的做法是用 L1 的私钥对一串固定的 message 做签字，这个动作是在 zksync 提供的界面进行，产生的这个签名就是使用者在 L2 的私钥。
    • 想法：这样一来，zksync 就可以直接确认该名使用者真的拥有特定的这个 L1 的私钥
  • zksync 要这么做的原因：因为主链的私钥只有 32 bytes，而 zksync 在 L2 用的新曲线（jubjub），私钥长度是 64 bytes。
    • 使用新曲线是为了提升安全性提及增加运算效率
  • 但是因为 L2 私钥的产生是在 zksync 的界面进行，如果攻击者将使用者带到一个长得很像但恶意的界面，使用者的私钥就会泄露。
  • 甚至可能使用者也会怀疑 zksync 有可能储存他们的私钥
